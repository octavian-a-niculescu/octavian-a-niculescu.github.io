Observatie - un BST are urmatoarele proprietati:
  • la mijloc se plaseaza un marcaj A
  • la sferturi un marcaj B
  • la optimi un marcaj C
  • iar la saisprezecemini un marcaj D
#include "iostream"
#include "string.h"
using namespace std;
void cerinta()
{   cout << "Avem nevoie de o rigla garadata speciala .Lungimea riglei este data de la tastatura ,L (deoarece se vor face impartiri succesive la 2 este indicat ca lungimea data sa fie o putere de 2 mai mare decat 16 pt un rezultat cat mai corect:64,128 etc), dar gradatiile nu trebuie sa marcheze milimetrii sau centimetrii, pe rigla gradatiile trebuie a fie astfel :";
	cout << "\n-la mijloc se plaseaza un marcaj A \n -la sferturi un marcaj B \n -la optimi un marcaj C \n -iar la saisprezecemini un marcaj  D\n";
}
void Gradare(int start, int capat, int contor, char marcaj, char v[1000])
{
	if (start < capat - 1 && contor>0)
	{
		v[(start + capat) / 2] = marcaj;
		Gradare(start, (start + capat) / 2, contor-1, marcaj+1, v);
		Gradare((start + capat) / 2, capat, contor-1, marcaj+1, v);
	}
}
int main()
{
	cerinta();
	int L, start, capat;
	char v[1000];
	cout << "Dati lungimea riglei \n";
	cin >> L;
	start = 0; capat = L - 1;
	for (int i = 0; i < L; i++)
		v[i] = '_';
	Gradare(start, capat, log2(16), 'A', v);
	for (int i = 0; i < L; i++)
		cout << v[i];
	return 0;
}
Explicatie rezolvare: Ideea de Divide et impera este de a utiliza o functie Gradare cu 5 parametrii : start reprezinta inceputul zonei ce trebuie marcata , capat sfarsitul acestei zone , contor reprezinta numarul de diviziuni de care avem nevoie , in cazul nostru saisprezecimi , marcaj este litera cu care va fi marcata partitia iar v este un vector de caractere , initial fiind doar o linie de L caractere. Daca capat>0 atunci se calculeaza mijlocul , se adauga la mijloc o gradatie , apoi se apeleaza recursiv functia Gradare pentru jumatatea din stanga , apoi pentru cea din dreapta , specificand pentru mijloc o noua litera cu care va fi marcata partitia. Am folosit un sir de caractere de maxim 1000 de caractere pentru a „desena” rigla , aceasta va fi initial „_” pe toata lungimea sirului. Solutia este asemanatoare algoritmului de cautare binara , la fiecare pas injumatatind domeniul de cautare.
Complexitate: Θ(nlogn)

Algoritmul este asemanator celui de Cautare Binara. La fiecre pas calculatorul genereaza un numar random (prin subprogramul GenerareNumar) care va fi luat ca pivot iar cautarea se va face in functie de compararea numarului gandit de utilizator cu pivotul ales random. Se introduc limitele Stanga si Dreapta initializate cu 1 si 100 intre care se va genera numarul random. Utilizatorul va introduce 'M' daca numarul oferit este mai mare decat cel la care s-a gandit acesta , daca este mai mic vei scrie 'm' iar daca este egal vei scrie '='. Folosim contorul Am ghicit initializat cu 0 pe post de conditie de opririe.Atunci cand utilizatorul introduce caracterul '=' Am ghicit va lua valoarea 1 , astfel oprind executia. Odata ce este introdusa alta valoare decat '=' limitele Stanga si Dreapta se vor modifica dupa caz:
Pasi executie:
  1. Daca este introdus 'm' contorul Stanga va lua valoarea numarului random generat+1 astfel cautarea facandu-se intre numarul respins anterori ca fiind prea mic si limita superioara, initial 100
  2. 2. Daca este introdus 'M' contorul Dreapta va lua valoarea numarului random generat-1 astfel cautarea facandu-se intre numarul respins anterori ca fiind prea mare si limita inferioara, initial 0.
