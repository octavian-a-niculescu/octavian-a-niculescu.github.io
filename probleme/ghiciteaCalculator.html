Tu si calculatorul puteti juca un joc impreuna: Tu te gandesti la un numar cuprins intre 1 si 100 iar celalalt incearca sa-l ghiceasca ,punand cat mai putine intrebari.
Daca numarul oferit este mai mare decat cel la care te ai gandit vei scrie'M' , daca este mai mic vei scrie 'm' iar daca este egal vei scrie '='. 
#include "iostream"
#include "stdlib.h"
#include "time.h"
using namespace std;
void cerinta()
{ cout << "Voi ghici un numar intre 0 si 100: daca raspund corect scrie '=' , daca numarul este prea mic scrie 'm' ,daca numarul este prea mare scrie 'M' \n";
}
int GenerareNumar (int min,int max)
{
    srand(time(0));
   return rand() % ((max - min) + 1)+min;
}
int main()
{
    cerinta();
    int incercari = 1, AmGhicit = 0, NumarGhicit, Stanga = 1, Dreapta = 100;
        char Raspuns;
        while (AmGhicit == 0 && Stanga<=Dreapta)
        { {cout <> Raspuns;
            if (Raspuns == '=')
            {
                AmGhicit = 1;
                cout << "Am ghicit , mi-a luat " << incercari << " incercari";
            }
            }
            if (Raspuns == 'm')
            {
                Stanga = NumarGhicit + 1;
                cout << "Voi cauta un numar mai mare , intre " << Stanga << " si " << Dreapta << "\n";
                incercari++;
            }

           else if (Raspuns == 'M')
              {
              Dreapta = NumarGhicit - 1;
              cout << "Voi cauta un numar mai mic , intre " << Stanga << " si " << Dreapta << "\n";
              incercari++;
              }
        }
    return 0;
}
Explicatie rezolvare: Impart problema in subprobleme asemanatoare , independente si de dimensiuni mai mici : aceeasi cautare dar pe intervale mai mici pana cand gasim rezultatul corect. Pentru a diminua timpul de cautare tinem cont de faptul ca elementele domeniului sunt ordonate (numere de la 1 la 100): jucatorul compara elementul x cu elementul generat de subprogram si introduce relatia dintre aceste 2 numere. Solutia este asemanatoare algoritmului de cautare binara . De data aceasta, calculatorul nestiind numarul , nu poate face o comparatie directa, continuarea algoritmului facandu-se in functie de raspunsul jucatorului . Dupa fiecare raspuns al jucatorului capetele domeniului de cautare vor fi actualizatare iar subprogramul „Generare Numar” va alege un numar la intamplare din acel interval. Daca numarul generat este prea mic atunci capatul din stanga al domeniului de cautare va fi actualizat cu numarul generat + 1 iar daca numarul este prea mare capatul din dreapta va deveni numarul generat - 1. In acest caz, rezolvarea problemei intiale se reduce la fiecare pas la rezolvarea unei singure subprobleme.
Complexitate: O(log n)
Demonstratie complexitate:
Daca ar fi sa implementam algoritmul clasic , am trece prin 100 de numere incarcand sa ghicim corect, ceea ce ar duce la aproximativ 100 de iteratii dar impartind problema in subprobleme simple si intervalul de valori in parti mai mici vom avea mult mai multe sanse sa ghicim numarul din cat mai putine incercari. Complexitatea devenind logaritmica. Sa analizam algoritmul in cazul cel mai defavorabil , adica cand numarul este depistat la ultimul apel.La fiecare apel spatiul de cautare se micsoreaza , deoarece numarul generat este aletoriu nu putem face un calcul exact dar presupunand ca se injumatateste atunci dupa primul apel ,din cele n elemente initiale , raman n/2 iar dupa al doilea n/4 elemente si tot asa. Din inductie am ajunge la afirmatica ca dupa k apeluri recursive spatiul va contine n/2^k elemente care , in cel mai rau caz , ar trebui sa fie 1. n/2^k = 1 => n=2^k => log2 n = k unde n este numar de elemente date initial , in cazul nostru 100.
Algoritmul este asemanator celui de Cautare Binara. La fiecre pas calculatorul genereaza un numar random (prin subprogramul GenerareNumar) care va fi luat ca pivot iar cautarea se va face in functie de compararea numarului gandit de utilizator cu pivotul ales random. Se introduc limitele Stanga si Dreapta initializate cu 1 si 100 intre care se va genera numarul random. Utilizatorul va introduce 'M' daca numarul oferit este mai mare decat cel la care s-a gandit acesta , daca este mai mic vei scrie 'm' iar daca este egal vei scrie '='. Folosim contorul Am ghicit initializat cu 0 pe post de conditie de opririe.Atunci cand utilizatorul introduce caracterul '=' Am ghicit va lua valoarea 1 , astfel oprind executia. Odata ce este introdusa alta valoare decat '=' limitele Stanga si Dreapta se vor modifica dupa caz:
Pasi executie:
  1. Daca este introdus 'm' contorul Stanga va lua valoarea numarului random generat+1 astfel cautarea facandu-se intre numarul respins anterori ca fiind prea mic si limita superioara, initial 100
  2. 2. Daca este introdus 'M' contorul Dreapta va lua valoarea numarului random generat-1 astfel cautarea facandu-se intre numarul respins anterori ca fiind prea mare si limita inferioara, initial 0.
