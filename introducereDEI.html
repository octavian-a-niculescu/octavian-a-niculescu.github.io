Intelegerea metodei DAC
Un algoritm de tipul Divide et Impera “sparge” problema principala in 2 sau mai multe sub-probleme similare sau de acelasi tip, pana cand una dintre acestea devine suficient de simpla incat poate fi rezolvata direct. Solutiile subproblemelor sunt ulterior combinate, astfel incat la final se gaseste solutia problemei originale. Tehnica Divide et Impera sta la baza multor algoritmi eficienti, cum ar fi sortarile, inmultirea numerelor mari, etc.
Etapele rezolvarii acestui tip de problema sunt:
  • (Divide) Descompunerea problemei initiale in subprobleme similar sau de acelasi tip, independente. Acestea la randul lor se descompun in alte subprobleme, pana cand devin suficient de simple ca sa poata fi rezolvate imediat.
  • (Impera) Rezolva subproblemele.
  • (Combina) Combinarea solutiilor subproblemelor, astfel incat sa se poata obtine problema initiala.
Avantaje
  1. Presupunand ca avem de efectuat un task foarte greu, divide et impera ne poate face munca mult mai usoara impartinand problema initiala in sub-probleme accesibile.
  2. Datorita faptului ca sub-probleme pot fi foarte usor de rezolvat pentru un calculator, divide et impera sta la baza multor algoritmi de sortare.
  3. Natura metodei divide et impera conduce la fenomenul de parallel processing.
  4. Complexitatea (timp) este scazuta.
  5. Daca sub-problemele sunt suficient de mici, acestea pot fi rezolvate in cache, fara accesarea memoriei principale (care e mai "greoaie").
Dezavantaje
  1. Descompunerea problemei in subprobleme poate fi foarte complexa ceea ce ar motiva abordarea unei alte metode de programare in afara de Divide et Impera in rezolvarea acesteia
  2. Natura recursiva a solutiei poate conduce la duplicarea sub-problemelor/rezultatelor.
    3. Varianta in care acestea sunt salvate static sau dinamic fiind mai viabila (de exemplu Fibonacci).
  3. Complexitatea (spatiu) este ridicata.
  4. Recursivitatea constanta in cazuri mici de baza poate duce la stive umplute ineficient din punct de vedere al memoriei, fiind mai desirabila o abordare a unor cazuri de baza mai mari.
Identificare DEI
Daca problema prezinta urmatoarele particularitati, se poate rezolva cu Divide et Impera.
  1. Se poate descompune în (doua sau mai multe) subprobleme.
  2. Aceste subprobleme sunt independente una fata de alta (subproblema nu se rezolva pe baza alteia si nu se foloseste rezultatul celeilalte).
  3. Aceste subprobleme sunt similare cu problema initiala.
  4. Problemele se pot descompune in alte subprobleme mai simple.
  5. Aceste subprobleme simple au solutii rapide.
De ce folosim DEI?
Daca o problema pe un set de date de intrare dat, poate fi aplicata direct (fara a redefini problema in sine) pe parti independente, ce nu se suprapun, a datelor de intrare, atunci problema poate fi rezolvata eficient folosind metoda Divide et Impera.
Daca imparti datele de intrare in parti ce se suprapun sau nu sunt independente, si aplici algoritmul DEI fiecarei astfel de parti, atunci ai executa de mai multe ori partea ce se suprapune. In astfel de cazuri, unde sub-problemele au elemente comune, folosim Programare Dinamica.
Exemplu: Cautati un stilou intr-o camera ce contine un pat, o masa, un scaun si un dulap.
Aici, sarcina principala este „cautarea stiloului”. Aceasta sarcina de „cautare” poate fi, fara a redefinii problema, aplicata independent drept „cauta stioul pe pat”, „cauta stiloul pe scaun”, „cauta stiloul pe masa”, „cauta stiloul in dulap”. Combinand rezultatele celor 4 sub-probleme putem spune daca stiloul se afla sau nu in camera.
Cand nu are sens sa folosim Divide et Impera:
Cand nu poti imparti datele de intrare in sub-parti independente
Exemplu: in cadrul aceleiasi probleme de mai sus, daca am "cauta" pe perna ce se afla in pat, aceasta sarcina s-ar suprapune celei de a "cauta" in pat. Daca facem asta, cautam daca pixul se afla pe perna de 2 ori! 
#include "iostream"
int factorial(int n)
{
if (n == 0)
{
return 1;
}
return n * factorial(n - 1);
}
                
int main()
{
 std::cout << factorial(5);
return 0;
}

Complexitatea de spatiu: o stiva este creata pentru a retine fiecare rezultat al apelului unei functii pana cand valoarea finala este returnata. Pentru exemplul nostru cu n-5, stiva retine urmatoarele valori: f(5) -> f(4)-> f(3) -> f(2) -> f(1) -> f(0). Putem observa astfel, ca pentru n=5, 5 elemente vor fi retinute de catre stiva. Coreland aceasta relatie pentru n aleatoriu, putem spune ca n elemente vor fi retinute de catre stiva in apel. Astel complexitatea este O(n).
Pasi (Divide):
  1. In main este apelata functia, cu n=5.
  2. Pt n=5, factorial returneaza 5*factorial(4). Astfel, este pus in stiva 5*factorial(4).
  3. Se calculeaza factorial(4), ce returneaza 4*factorial(3), urmatorul element din stiva devenind 4*factorial(3).
  4. Se calculeaza factorial(3), ce returneaza 3*factorial(2), care este pus si in stiva.
  5. Factorial(2) este 2*factorial(1) – se pune in continuare in stiva.
  6. Factorial(1) este 1*factorial(0) – ultimul element pus in stiva.
  7. Divizarea se opreste, deoarece factorial(0) este 1.

Urmatorul pas (Impera):
  1. Factorial(0) este 1, deci 1*factorial(0) este 1.
  2. 2*factorial(1) este 2*1=2.
  3. 3*factorial(2) este 3*2=6
  4. 4*factorial(3) este 4*6=24
  5. 5*factorial(4) este 5*24=120 <- care este problema initiala, deci raspunsul este 120.
